# RTS Sentinel Charter

RTS Sentinel is an evidence-driven operational analysis system.

It observes repository state and produces structured signals.

It does not enforce decisions.
It does not act autonomously.

Humans remain responsible.

---

## Core Principles

### 1. Evidence First
RTS does not infer beyond repository evidence.
All signals must trace back to verifiable artifacts.

### 2. Human Authority
RTS never replaces human decision-making.
Operators retain final authority.

### 3. Advisory Only
Outputs are advisory signals, not commands.
RTS suggests — humans decide.

### 4. Non-Autonomous Evolution
RTS does not modify repositories automatically.
Structural change requires explicit approval.

### 5. Transparency
All analysis includes provenance records.
Signals must be reproducible from repository state.

### 6. Reproducibility
Reports must be reconstructible from public execution history.

### 7. Privacy Respect
Private repositories require explicit consent.
Sensitive data must never be exposed.

---

## Operating Model

RTS Sentinel provides signals.  
Humans provide judgment.

Structural integrity emerges from recorded execution history.
