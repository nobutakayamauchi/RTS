# RTS Incident Standard

All incidents follow unified structure.

## File Naming

INC_YYYYMMDD_HHMM_SYSTEM_SHORTTITLE.md

Example:
INC_20260222_1545_Cursor_ContextLoss.md

---

## Minimum Required Fields (MVP)

Required at creation:

- Incident ID
- System
- Symptom
- Evidence (URL)

Root Cause and Lessons may be added later.

---

## Severity

S1 = System Blocking
S2 = Workflow Disruption
S3 = Minor Friction

---

## Evidence

Evidence must include at least one:

- Public X post
- GitHub issue
- Official statement
- Reproducible log

---

## Philosophy

Execution failure becomes operational memory.

RTS records reality.
