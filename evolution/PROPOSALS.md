# RTS Evolution Proposals

This file records all proposed structural or behavioral changes to this RTS instance.

Rules:

1. RTS may propose improvements.
2. RTS may NOT self-apply structural changes.
3. Only the originator may approve changes.
4. Approval must include explicit reasoning.
5. All approved changes must be logged.

Format:

No proposal may execute without approval.
