## Context
近年、災害対応や有事対応において  
「なぜその判断をしたのか」「なぜ別の対応を取らなかったのか」  
が、事後的に問われるケースが増加している。

しかし実際の現場では、  
- 情報が不完全  
- 時間が不足している  
- 正解が事前に分からない  

という条件下で判断が行われており、  
その**判断プロセス自体が記録されない**ことが、  
後の責任論・萎縮・判断停止を招いていると考えた。

中国レアアース問題をはじめとする  
「コストや責任を踏み倒す側が主導権を握り、  
　現場で判断する側が不利になる社会構造」への問題意識から、  
判断と不判断を記録・可視化する仕組みとして  
RTS（Real-Time Trust System）を構想した。

宮古島は、  
- 離島特有の制約条件  
- ブラックアウト時の通信・医療・物資の脆弱性  
- 防災・有事対応の実務が集約されやすい環境  

を併せ持ち、  
この構造的課題を検証する**社会的・地理的に圧縮されたモデルケース**として
適していると判断した。

## Objective
- 有事対応における「判断プロセスの記録」という考え方について  
  防災実務の現場から意見を得ること
- 新規設備や制度導入を前提とせず、  
  現場負荷を増やさない形での検討可能性を探ること
- 一方的な提案ではなく、対話の入口を作ること

## Options Considered
- Option A: 学術機関・大学から先行してアプローチする  
- Option B: 県・国レベルの防災機関へ先に提案する  
- Option C: 現場に最も近い市町村防災担当へ直接連絡する  

## Decision / Action Taken
Option C を選択。

宮古島市の公式問い合わせフォームを通じて、  
防災担当部署宛に以下を送付した。

- 有事対応における判断プロセス記録の考え方を整理した資料
- 本構想が責任追及や設備導入を目的としないことの明示
- 意見交換・情報共有を目的とした連絡である旨の説明

面談や現地訪問は前提とせず、  
まずは文面・資料による意見交換を希望する形とした。

## Rationale
- 有事対応において最も判断負荷を負うのは市町村の現場である
- 上位機関からの提案ではなく、  
  現場起点での検討余地を残すことが重要だと判断した
- 初期段階で「何かを導入させる」印象を与えることを避け、  
  防災実務の視点を尊重するため、問い合わせ形式を選択した

## Known Risks / Unknowns
- フォーム経由のため、担当部署に届かない可能性
- 構想の意図が誤解される可能性
- 返信が得られない可能性
